# Cacti OS

A personal knowledge and task system for Claude Code. Clone it, run the onboarding interview, and Claude knows who you are, what you're working on, and what you care about — every session.

This is the personal system I built and use daily, packaged as a template.

## What you get

- **Persistent context** — Claude reads your `HANDOVER.md` at every session start. It knows where you left off without you explaining.
- **Handover archiving** — Run `/compact` and Claude updates HANDOVER.md before compressing. Resolved threads archive to `handover/` weekly files. Your context stays clean and accurate across sessions.
- **Memory** — Claude builds a `memory/` of topic files as you work: who you are, preferences, lessons, references. It grows silently every session.
- **Life OS** — A structured place for your goals, decisions, and projects. Claude references it when you're making decisions, not just when you ask.
- **Monitoring agents** — Scheduled Claude instances that watch whatever domains matter to you. One generic template included — copy and customize.
- **Skills** — `/today`, `/quick-task`, `/log`, `/weekly-review`, pre-wired and ready.
- **Self-improving** — Claude adds to your registry, contacts, and knowledge files as you work.

## Prerequisites

- [Claude Code](https://claude.ai/code) installed
- Git
- macOS (scripts use bash; should work on Linux with minor edits)

## Install

```bash
git clone https://github.com/tiobenito/cacti-os.git ~/cacti
cd ~/cacti
bash scripts/setup.sh
```

Then open Claude Code from `~/cacti/` and run `/onboarding`. The interview takes about 10 minutes. Afterwards, your system is live.

## Day 1 experience

After `/onboarding` completes:

- `CLAUDE.md` knows your name, role, and primary projects
- `knowledge/registry.md` maps your project folders to aliases ("work on the newsletter" routes Claude to the right directory)
- `life/goals/2026.md` has your goals seeded from the interview
- `knowledge/contacts.md` has your key people
- Running `/today` gives you a morning briefing

Over the next week:
- Claude adds projects to the registry as you mention them
- `memory/` accumulates topic files about you and your preferences
- `HANDOVER.md` builds up your real active threads

## How context continuity works

Run `/compact` when your conversation gets long. Before compressing, Claude:

1. Updates `HANDOVER.md` with the current thread status, any new decisions, and a session log entry
2. Archives resolved threads to `handover/YYYY-MM-DD.md` (weekly files, never deleted)
3. Compiles new knowledge into the right files (contacts, goals, decisions)

Next session, Claude reads HANDOVER.md at start and picks up exactly where you left off.

## System map

```
~/cacti/
├── CLAUDE.md           The brain — how Claude behaves as your chief of staff
├── HANDOVER.md         Active context — open threads, triage, session log
├── tasks.md            Quick-capture inbox
├── memory/             Persistent topic files (MEMORY.md is the index)
├── handover/           Weekly archive of resolved threads and old session logs
├── knowledge/          Routing map, goals, contacts, calendar
├── life/               Who you are, what you want, major decisions
├── agents/             Scheduled monitors
├── daily/              Daily briefing files (auto-generated by /today)
├── scripts/            Bootstrap and utility scripts
└── .claude/
    ├── commands/       /handover command (runs before /compact)
    └── skills/         Skill library
```

Three layers:
1. **Cacti** (this repo) — strategy, knowledge, cross-project context
2. **Projects** (your code folders) — each gets a `CLAUDE.md` for project-specific context
3. **Global** (`~/.claude/`) — Claude Code preferences, personal skills

## Skills

| Skill | What it does |
|-------|-------------|
| `/onboarding` | First-run interview — populates knowledge files |
| `/today` | Morning briefing from HANDOVER, tasks, calendar, inbox |
| `/quick-task` | Fast task capture ("remind me to call X") |
| `/log` | Log a note, meeting, or event |
| `/weekly-review` | Weekly synthesis — what happened, what's next |

## MCP setup

MCPs connect Claude to your actual tools (GitHub, Gmail, Slack, Notion, etc.). The `.mcp.json` in this repo has commented-out configs for common servers — uncomment what you use.

For setup instructions: [modelcontextprotocol.io](https://modelcontextprotocol.io)

The target user for this system is comfortable setting up MCP servers. If you're not there yet, the system works fine without them — it just briefs from local files.

## Customization

See `docs/customizing.md` for:
- Adding new projects to the registry
- Creating domain agents (work health, fitness, finances)
- Writing your own skills
- Scheduling agents to run automatically

## Philosophy

This is a markdown-first, git-tracked, local-first system. No database. No SaaS dependency. No subscription. It runs entirely in Claude Code with files you own and control.

The structure is intentionally opinionated. The content is yours.
